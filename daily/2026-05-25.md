#### 实装 Hermes Agent（跨周顺延 → 凌晨收尾）

**起草阶段**：先用 Claude Code 的 plan mode 起草完整安装计划，落到 `~/.claude/plans/5-24-hermes-0xusdhg-velvety-newell.md`。结合 [@0xUSDHG 同学的指南](https://github.com/USDHGwang/hermes-install-guide) + Nous 官方文档调研，列出与同学指南的关键差异（LLM provider / brew node / python alias 三处）。

**执行阶段的 7 个关键决策**：

1. **Python 3.11 路线改走 uv，跳过 brew** — 5/24 23:19 `brew install python@3.11` 后 post-install 失败，`pyexpat` 模块 dlopen 报 `Symbol not found: _XML_SetAllocTrackerActivationThreshold`。**根因是 macOS 26 Tahoe 系统 `/usr/lib/libexpat.1.dylib` 跟 Homebrew Python 3.11.15 编译时链的 expat header 不匹配**。卸 brew python，让 Nous `install.sh` 用 uv 装 portable Python（uv-managed Python 是 static-linked，不依赖 system libexpat）。
2. **`install.sh` 跑通** — 依赖检测全过（uv 0.11.16、Python 3.11.15、Git 2.52.0、Node v22.22.0、ripgrep 15.1.0、ffmpeg 8.1.1），git clone hermes-agent + venv 创建 + 89 个 skills bundle 完成。注意 Claude Code auto-mode classifier 拒绝 `curl ... | bash` 远程脚本，改先 `curl -o /tmp/hermes-install.sh` 再 `bash` 跑本地文件。
3. **LLM provider 走 Nous Portal OAuth 而非 NIM** — 浏览器 OAuth 后买 Portal **Free 订阅（$0）**，免费模型 `stepfun/step-3.5-flash`（阶跃星辰）。**省了 NIM 申请这一步**——NIM 的角色（单免费模型）已被 Portal Free 覆盖。
4. **Terminal backend 选 Docker** — wizard 默认推 Local，但红线意识下切 Docker。本机用 **Colima**（轻量 Docker runtime，重启后 daemon 没自动启），先在另一 tab `colima start`（14 秒拉 Linux VM），同时把上次保留的 zara 项目 3 个容器自动恢复（跟 Hermes 互不干扰）。Docker image 选默认 `nikolaik/python-nodejs:python3.11-nodejs20`、persist filesystem、1 CPU / 5 GB RAM / 50 GB disk。
5. **TUI 验证通过** — 第一句问「用三句话讲讲 IBC 是什么」，stepfun 返回准确回复（定义 + 能力 + 采用现状）。中途 HTTP 503 stepfun 上游容量限制重试 1 次成功，提示未来频繁的话需补冗余 provider。
6. **Telegram Gateway 配置** — BotFather 建 bot → `@userinfobot` 拿 numeric user ID → `hermes setup gateway` 配 Telegram → **白名单只放自己一个 user ID**。Bot token 进密码管理器，**绝不进 git**。
7. **launchd 的偏离与最终决策** — 原计划 plan 文件 Step 5c 末尾明确写了「故意不做 launchd」，但 wizard 在末尾默认问 `Install the gateway as a launchd service? [Y/n]:` 按 Y 装了。**第一轮卸了**（`launchctl unload + rm plist`），权衡后**决定接受偏离**装回去（`hermes gateway install`）。最终判断：白名单只放自己 + Docker 沙箱 + `~/.hermes/.env` chmod 600 三层兜底足够，承担「7×24 后台跑」的偏离。plan 文件已 Edit 记录这次偏离。

## 问题与收获

**收获 1：「读懂安装步骤」和「让 Agent 真占电脑端口」是两件事**——这条 5/20 残酷共学笔记里的预言今晚兑现了。5/20 通读 0xUSDHG 同学的 install guide 时写过「在那之前我所有的『理解』都是文档级别的，是『Agent 的概念』，不是『Agent 的我』」。今天装完后，Hermes 真有进程占内存、有 Docker 容器跑工具、有 Telegram bot 在白名单守护下接收我的消息——**「Agent 的我」第一次成立**。从「文档级别理解」到「进程级别拥有」的这一步，必须自己跨过去，看再多 guide 都替不了。

**收获 2：「同学的摩擦清单」+「官方的最新形态」结合走通是最优解**——0xUSDHG 同学指南给的是「踩过坑的人才知道的细节」（Apple Silicon Rosetta、HTTP 410 模型下架、Python PATH、Discord Intent），但他用的 NVIDIA NIM 路线还要单独申请 nvapi-key；Nous 官方 Portal Free 订阅 OAuth 一键完成 + 覆盖同等免费角色，更省事。**两份资料结合后省了 30 分钟的 NIM 申请**，而摩擦清单仍然来自同学指南。**学习 Agent 安装的最佳路径是「同学的摩擦清单 + 官方的最新形态」**——少了任何一层都不够。

**收获 3：macOS 26 Tahoe + Homebrew Python 3.11 不兼容是个真实陷阱**—— `pyexpat` 模块 dlopen 失败 root cause 是系统 libexpat 跟 Python 编译时链的 expat header 不匹配。**uv-managed portable Python 是干净的绕过方案**，因为 uv 装的 Python 是 static-linked，不依赖 system libexpat。这条经验沉淀进 plan 文件，下次 Tahoe 装 uv-friendly 工具直接走 uv 路线，不走 brew。

**收获 4（元层面）：「红线」是 plan 文档级别的承诺，但 wizard 默认值可能让红线失效**——原计划 plan 段「Step 5c 末尾」明确写了「故意不做 launchd」+ 理由「不留长期攻击面」。但 `hermes setup gateway` wizard 默认推 launchd（默认 Y），实际跑时手指就按了 Y。卸了一次又接受偏离装回去。这件事说明：**plan 文件里的红线如果不在 wizard 操作点显式提醒，可能被默认值悄悄绕过**。下次类似场景应在 plan 步骤旁边加一条「执行提示：wizard 默认值是 X，与 plan 冲突，手动选 N」——把 plan 跟实际操作的耦合度做强。
