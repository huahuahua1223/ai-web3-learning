### 阅读 / 观看

#### 5.27 直播｜Neo-Cypherpunk & the Cultural Layers of Privacy

这场直播对我最大的推动，不是又多知道一个 privacy 口号，而是把我这两天一直在想的「Agent 边界」问题补完整了。

5/26 看 Cobo 时，我更在意的是 **permission boundary**：Agent 一旦要碰钱包、预算、链上动作，授权必须具体到任务、额度、时间窗、失败处理。5/27 Peter Farbey 这场又补了另一层：**就算授权边界想清楚了，隐私边界也不会自动成立。**

一个 Agent 能不能替用户做事，和一个 Agent 默认暴露多少信息，其实不是同一个问题。它跟谁通信、哪些上下文会被带出去、哪些行为轨迹会被默认公开、什么地方应该先保护用户不被观察或不被诈骗，这些都应该是产品设计前置考虑，而不是事后补洞。

Peter 讲隐私不是 0 和 100、不是「要么全做要么别做」这一点，我觉得尤其重要。它把隐私从一种很高门槛、很 hardcore 的身份要求，拉回到一种可以渐进落实的产品和行为选择。这也让我更明确：Week 2 如果真的要把 Agent 放进 Web3 的真实使用场景，**隐私不是边角料，而是跟 permission strategy 平级的一层约束。**

## 问题与收获

**收获 1：权限边界和隐私边界不是一个问题的不同说法，而是两层独立约束。**
前一天 Cobo 把「Agent 碰钱时怎么被限制」这层讲清楚了；今天 Peter 又让我更明确：**被限制得足够细，不等于被保护得足够好。** 一个 Agent 可以只执行白名单动作，但仍然可能默认暴露过多信息。

**收获 2：一个更成熟的 Agent，不一定是更能干，而可能是更会默认保护人。**
以前我更容易把“更好的 Agent”理解成接更多工具、自动化更多步骤、替我做更多事。现在开始觉得，真正有区分度的地方可能是：它是不是默认少拿一点权限、少暴露一点上下文、把撤回和审计做得更简单。

**问题 1：这两场直播带来的判断已经收敛出来了，但还没转成文档产物。**
`restricted-web3-assistant-design.md` 目前还停在 5/19 的版本；现在至少要补两层：一层是 task-level authorization / budget / expiry / audit trail，另一层是 privacy layer / default visibility / communication exposure。认知已经到位，产物还没跟上。
