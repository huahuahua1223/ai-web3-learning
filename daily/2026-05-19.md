### WCB Agent API 探索

启动 Prompt 第 7 节提到 WCB Agent API，今天第一次真正用上：

- API 文档（machine-readable）：<https://web3career.build/llms.txt>
- 入口：`POST https://web3career.build/api/agent/call`，所有调用走一个 endpoint，body 用 `{ procedure, input }`（tRPC 风格）
- 认证：profile 设置里生成 Secret API Key，用 `Authorization: Bearer <KEY>` 发送
- Key 安全存储用 macOS Keychain（`security add-generic-password`），Claude Code 调用时从 keychain 现取，永远不进任何文件 / 环境变量持久化
- 学员侧能直接用的 procedure：`users.getProfile` / `tasks.listForLearner` / `tasks.myTaskHistory` / `tasks.submitEvidence` / `tasks.myTotalPoints` 等

### 关键收获

- **LLM 训练数据陈旧 vs 真机验证**：Plan agent 给的代码用了 Etherscan V1 接口（训练数据里 V1 还活着），真跑才发现 V2 已强制。这是「AI 协作开发」里最典型的人工介入点——LLM 写得快，但**真实接口/版本必须真机跑一次才能确认**。这条已经写进任务证明文档第 5 节人工复核记录。
- **浏览器直调 LLM 的工程边界**：Anthropic 必须带 `anthropic-dangerous-direct-browser-access: true` header；OpenAI 在 401 时不返回 CORS header → 浏览器层面表现为「网络错误」，让用户在「Key 错」和「网络问题」之间瞎猜。错误提示工程化对纯前端 LLM 应用非常关键。
- **中转/聚合 API 的价值**：智增增这类聚合服务用 OpenAI 兼容协议托底，CORS preflight 通过、payload 不用改、模型一行配置切换，对中国大陆开发者是几乎零成本的解药。配合美团 LongCat、DeepSeek 等免费/廉价模型，Hackathon 跑大量请求的成本能压到接近零。
- **WCB Agent API 是一等公民**：平台把所有 procedure 用 tRPC 暴露 + 提供 `llms.txt` 让 AI Agent 直接读取，比传统 REST + Swagger 对 Agent 友好得多。未来可以做：每日打卡自动提交、新任务推送、积分自动汇总、Handbook feedback 自动入库。

## 问题与收获

- 浏览器 ES module 的强缓存：`script.js?v=N` 这个版本号只让 script.js 本身刷新，但它 `import` 的 `prompts.js` 没带 query，浏览器仍用旧版。修复：`import {...} from "./prompts.js?v=N"`，依赖文件一起 bump。
- WCB API 的 `tasks.listForLearner` 在不传 `trackId` 时返回空数组；`tracks.listForProgram` 又对 agent 关闭。最后绕路：从 `program.getById` 返回的 `metadata.taskI18n.en` 里拿到全部 38 个 taskId，再挨个调 `tasks.myTaskHistory(taskId)` 拼出进度报告。如果 trackId 后续要用，可能得在网页端 inspect 或问平台运营。
- macOS Keychain 在新 shell 第一次读会弹授权对话框，选「始终允许」之后免打扰；Claude Code 的 Bash 工具开新 shell 不会 source `~/.zshrc`，所以 env 变量取不到，但直接 `security find-generic-password` 能读到 keychain——反而是 keychain 路径更便携。
